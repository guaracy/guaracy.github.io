1 Introdução

O objetivo deste documento é gerar um arquivo de configuração para o Emacs utilizando Literate Programming, isto é, iremos escrever a documentação e o código e depois o código será extraído automaticamente gerando o arquivo init.el.

É a configuração que eu estou utilizando no momento. Quaisquer alterações serão feitas neste documento e um novo arquivo init.el será gerado após as alterações. O arquivo ~/.emacs.d/init.el será avaliado quando o emacs iniciar. Ao pressionar Ctrl+c Ctrl+v t, todos os blocos deste documento serão gravados no arquivo init.el. Em configurações manuais, pode ser interessante colocar configurações complexas em arquivos separados, o que não seria difícil de gerar em um documento .org.

Esta configuração não é para os usuários do Vim (pelo menos no início). Se você é um feliz usuário do Vim e deseja aprender Emacs para usar suas facilidades como o Org-mode, etc., você deverá visitar o Spacemacs, Spacelite, Doom e outros. Eles configuram o Emacs de forma que você se sinta mais a vontade com relação aos atalhos do Vim e/ou visualmente.

De qualquer forma, existem mais de 3.600 pacotes para você configurar seu Emacs para que fique mais adequado para o que você vai fazer. O melhor é você procurar na internet o que ou como os usuários configuraram o ambiente para uma determinada finalidade. Se você trabalha com Ruby, pode verificar como este usuário configurou o ambiente dele. Ou uma configuração mais específica para PHP ou para Python, Elixir, Haskell, etc.. É a melhor configuração para ele mas já é um bom começo. No site sa Sacha sempre tem as novidades da semana que incluem pacotes, dicas, etc.. O melhor pode ser começar com alguma configuração existente, procurar por outras e ir adaptando para o seu gosto pessoal.

2 Geração da configuração

Poderíamos informar os pacotes desejados, carregar todos os necessários e efetuarmos as configurações individuais em tópicos específicos para cada um. Usarei o use-package para manter a maioria da configuração dos pacotes agrupadas (menos as configurações para o org-mode que deixei para o final).

Para não gerar um determinado código para a configuração do pacote, utiliza-se :tangle no.

3 Arquivo init.el

O que segue é o conteúdo disponibilizado para o arquivo init.el.

Iremos definir tema, tamanho da janela e outros aspectos visuais.

11:         (height . 40)))

18: (setq initial-scratch-message nil)

3.3 Instalação dos pacotes

Para gerenciar as configurações dos pacotes iremos utilizar o use-package para deixar as coisas mais ordenadas. Não que seja necessário pois o arquivo será gerado automaticamente por este documento e, qualquer alteração, deverá ser feita por nele. Os pacotes que não estão instalados (caso algum seja incluído depois) serão automaticamente instalados e configurados pelo use-package. 

38: (unless (package-installed-p 'use-package)
39:   (package-refresh-contents)
40:   (package-install 'use-package))
41: (require 'use-package)

Algumas opções do use-package:

  • :ensure t → indica que o pacote deverá ser baixado se não existir localmente
  • :init → configurações antes do pacote ser carregado
  • :config → configuração após o pacote ser carregado
  • :bind → criar um atalho para uma função
  • :disabled → desabilita o pacote

3.3.1 which-key

Mostra os atalhos disponíveis quando o usuário digita Ctrl-x, Ctrl-c, Ctrl-h, etc. e não digita imediatamente a sequência. 

42: (use-package which-key
43:   :ensure t
44:   :config
45:   (which-key-mode)
46:   (setq which-key-idlw-delay 0.5))

3.3.2 smex

Facilita a execução de comandos quando o usuário digita Alt-x (comandos gerais) ou Alt-X (comandos do modo atual do buffer). 

47: (use-package smex
48:   :ensure t
49:   :bind (("M-x" . smex)
50:          ("M-X" . smex-major-mode-commands)))

3.3.3 iedit

Parecido com multiple-cursors, permite a edição simultânea de multiplas regiões. Pessoalmente, acho interessante o fato de C-' esconder o resto do texto para uma vizualização geral das ocorrências, editar apenas as ocorrências de uma função ou linha. 

51: (use-package iedit
52:   :ensure t
53:   :bind (("C-;" . iedit-mode)
54:          ("C-RET" . iedit-rectangle-mode)))

3.3.4 multiple-cursors

Trabalha com multiplos cursores. 

55: (use-package multiple-cursors
56:   :ensure t
57:   :bind (("C->" . mc/mark-next-like-this)
58:          ("C-<" . mc/mark-previous-like-this)))

Permite exportar o conteúdo de um buffer para uma página .html com o mesmo tema utilizado pelo emacs bem como salientar a sintaxe dos fontes no org-mode.

3.3.18 rainbow-delimiters

Optei por utilizar apenas duas cores utilizando as cores do tema para keyword e string. 

108: (use-package rainbow-delimiters
109:   :ensure t
110:   :config
111:   (add-hook 'prog-mode-hook #'rainbow-delimiters-mode)
112:   (require 'color)
113:   (defvar my-paren-dual-colors
114:     '((face-attribute 'font-lock-string-face :foreground)
115:       (face-attribute 'font-lock-keyword-face :foreground)))
116:   (setq rainbow-delimiters-outermost-only-face-count 0)
117:   (setq rainbow-delimiters-max-face-count 2)
118:   (set-face-foreground 'rainbow-delimiters-depth-1-face
119:                        (eval (elt my-paren-dual-colors 1)))
120:   (set-face-foreground 'rainbow-delimiters-depth-2-face
121:                        (eval (elt my-paren-dual-colors 0)))
122:                        )

Você pode deixar cada fechamento de delimitar de uma cor. Substitua da linha 113 até a linha 121 pelo código abaixo: 

(cl-loop
 for index from 1 to rainbow-delimiters-max-face-count
 do
 (let ((face (intern (format "rainbow-delimiters-depth-%d-face" index))))
   (cl-callf color-saturate-name (face-foreground face) 30)))

4 Dicas

Se for exportar para html C-e h e aparece a mensagem Wrong type argument: listp, … pode ser suficiente digitar:

C-u M-x org-reload

Se aparecer o erro: Invalid function: org-babel-header-args-safe-fn rodar o código abaixo 

(byte-recompile-file
     (expand-file-name "ob-R.el"
                      (file-name-directory (locate-library "org")))
     t)
